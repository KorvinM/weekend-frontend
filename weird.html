Summary

Engine

  • JS code is interpreted and run by the JavaScript Engine, executed synchronously
  • engine creates a LIFO stack of execution contexts known as the call stack
  • 1 global context, and new contexts created by function calls. See https://youtu.be/8aGhZQkoFbQ?t=4m58s
  • each context has a creation and execution phase, and contexts are popped off the top of the stack when they are finished

Objects

A collection of name/value pairs is 'the simplest definition of an object' in js. Values can be further, nested name/value pairs

Scope

Understanding the execution context stack and variable environments helps enourmously in grokking the scope chain in js

Asynchronous events

  • Alongside the synchronous execution we have the event queue
  • The js engine waits for the execution stack to empty before checking the event queue, periodically

Primitives

  • undefined
  • null
  • Boolean
  • Number
  • String
  • Symbol (ES6)

Operators

Operators are actually functions using the infix syntax

Operator precedence and associativity

Use the strict comparison operators, === and !==, to avoid coercion. See Equality comparisons and sameness
